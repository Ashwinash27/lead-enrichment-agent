The pipeline

Benchmarked across 15 eval cases. Average end-to-end: 43.8s. Fastest: 18.9s. Slowest: 100.4s (common name disambiguation).

Phase A 5–8s

Request hits the API. Claude plans search queries while GitHub, news, and community tools fire simultaneously via asyncio.gather(). The planner runs alongside deterministic tools — it doesn't block.

Phase B 6–10s

Planner output gates this phase. It decides which Google queries to run and which URLs to scrape with Playwright. Up to 5 searches, 3 browser scrapes — all concurrent.

Phase B.5 2–5s

Email waterfall. Five layers, cheapest first: GitHub commit emails, regex scan, SMTP verify, Prospeo, Hunter.io. Most lookups resolve before touching paid APIs.

Phase C 22–29s

All results feed into Claude. One call extracts structured data, one generates use-case-specific talking points. This is the bottleneck — two LLM calls processing up to 20k chars of context.

Built on LangGraph with a StateGraph orchestrator. Each phase fans out concurrently with fallback at every layer.

Design decisions

Parallel planner
The AI planner runs alongside deterministic tools. Its output only gates Phase B — cutting total latency ~40% vs plan-then-execute.
5-layer email waterfall
Three free methods exhaust before spending API credits. Prospeo (75/month) and Hunter (25/month) are last resort.
Semantic vector cache
Qdrant embeddings at 0.92 cosine threshold. Near-duplicate requests resolve in <500ms instead of ~44 seconds.
Context truncation with retry
Extractor gets 20k chars max (3k per tool). If parsing fails, retry with 30k chars and 4.5k per tool.
